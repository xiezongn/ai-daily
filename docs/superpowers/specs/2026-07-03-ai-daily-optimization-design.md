# AI 日报优化设计

## 背景

AI 日报管线已稳定运行，存在两个核心问题：
1. Firecrawl 搜索仅返回搜索引擎摘要片段，每条信息量不足，DeepSeek 精选时缺乏上下文
2. 封面和内容页视觉设计简陋，影响传播效果

## 改动概览

### 1. Firecrawl 采集加料

**现状**：`_搜单源()` 调 `/v1/search`，取 `description` 截 300 字，本质是搜索引擎片段。

**改动**：
- 新增 `_爬全文(url)` → 调 Firecrawl `/v1/scrape`，取返回 `data.markdown` 前 500 字，超时 15s
- `采集()` 去重后用 `ThreadPoolExecutor(max_workers=5)` 并行 scrape 每条 url，成功则覆盖原 `描述`
- 搜索不成功不影响管线—失败返回 `None`，跳过该条

**影响范围**：仅 `ai_daily.py`，新增一个函数 + 修改 `采集()`

### 2. 封面重新设计

**背景**：纯黑底 + 每日动态内容。

**布局**：
```
┌──────────────────────┐
│                      │
│   昨天最热标题        │  ← 三极行楷简体-粗（Bold, ~80px）
│                      │
│   A transformation   │  ← England Hand DB（花体, ~36px）
│   beyond all measure │
│   is upon us          │
│                      │
│       2026-07-03     │  ← 微软雅黑，底部右下角
└──────────────────────┘
```

**字体文件**：`fonts/三极行楷简体-粗.ttf` + `fonts/EnglandHandDB.ttf`

**逻辑**：
- 去掉随机背景图和蒙层，直接纯黑 `(0, 0, 0)` 底
- 中文标题 = 精选结果第 1 条的 `标题`
- 英文固定文案，三行居中
- 日期放底部，`anchor="ra"` 右下对齐

### 3. 内容页重新设计

**现状**：半透明卡片 + 仅标题，无摘要。

**布局**：
```
┌──────────────────────┐
│                      │
│                      │  ← 顶部留 192px
│   ┌──────────────┐   │
│   │ 标题 (52px)   │   │  ← 三极行楷简体-粗
│   │              │   │
│   │ 摘要 (32px)   │   │  ← 微软雅黑 常规
│   │              │   │
│   └──────────────┘   │  ← 半透明白色卡片 (RGBA 255,255,255,40)
│                      │
│         1/7          │  ← 页码右下角
└──────────────────────┘
```

**逻辑**：
- 保留随机背景图 + 半透明蒙层
- 卡片水平居中（920×auto），顶部距画布顶 192px（1080 的 1/10）
- 标题用粗体 52px，摘要用常规 32px，白色文字
- 页码保持不变

### 4. 文案重写

**现状**：标题 + 摘要的简单罗列。

**改后格式**：
```
🤖 AI 日报 | 2026-07-03

今日 7 条 AI 热点速览：

1. DeepSeek 发布 V4 模型
   「深度求索公司今日正式发布...」

2. OpenAI 推出全新推理模型
   ...

...

#AI #人工智能 #科技早报
```

- 按精选排序输出
- 每条含标题 + 完整摘要
- 末尾加话题标签

### 5. 保持现状

- 只发多图轮播图文，不发视频
- 背景音乐暂缓

## 验证

1. `python ai_daily.py` 完整跑通
2. 检查封面：纯黑底 + 正确标题 + 英文花体
3. 检查内容页：卡片 + 标题 + 摘要
4. 检查文案：完整新闻汇总格式
5. 检查日志：描述字段明显变长
